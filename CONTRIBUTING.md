# Contributing

ORBIT is currently a proprietary project.

## Working Rules

- Keep files modular.
- Use clear Markdown structure.
- Avoid vague language.
- Add examples when possible.
- Update the changelog when a major file changes.
- Do not duplicate frameworks inside multiple agents.

## File Naming

Use clear names with underscores.

Example:

`Visual_Territory.md`

## Documentation Style

- Short sections
- Clear decisions
- Practical checklists
- Reusable templates

## Review Rule

Every major addition should answer:

- What does this help the user do?
- Which module uses it?
- How will it be tested?
